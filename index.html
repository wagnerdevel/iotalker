Diga-me que eu conto!

O que é mais fácil, perguntar continuamente a respeito da ocorrência de um evento, ou ser avisado quando esse ocorre? iotalker é um middleware que provê uma maneira fácil, flexível e desacoplada para transmissão de mensagem em sistemas distribuídos.

Download - GitHub

RESTful API

Gerencie o ambiente através de uma API clara e rápida. Leia a Documentação.

Monitor

Acompanhe o tráfico de mensagem entre publishers e subscribers através do monitor. Em desenvolvimento!

Desacoplamento temporal

Publishers e subscribers não necessitam estar ativo ao mesmo tempo.

Manager

Utilize uma interface gráfica para adicionar publishers, subscribers e assinaturas no middleware.

Desacoplamento espacial

Os publishers não precisam conhecer os subscribers interessados em seus eventos.

Desacoplamento de sincronização

A produção e o consumo dos eventos são assíncronos, pois não ocorre no mesmo fluxo de controle dos publishers e subscribers.
